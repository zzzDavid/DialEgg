# DialEgg & Extraction-Gym Bug Fix Batch: Decoupled Extraction Pipeline Hardening

**Date:** February 25, 2026  
**Goal:** Fix 9 reported bugs across DialEgg (C++) and Extraction-Gym (Rust) found during CIRCT hardware optimization runs  
**Status:** All 9 tickets resolved and archived  
**Commits:** `ea32425` (dialegg), `924991b` (extraction-gym)

## Executive Summary

Resolved a batch of 9 bug tickets filed against the Aspen hardware optimization pipeline. Five bugs required code changes to the `egg-opt` tool (DialEgg) and the `extract_result` binary (Extraction-Gym). Four bugs were traced to incomplete egglog rule definitions (`.egg` files) and required no tool changes. The fixes span three major areas: S-expression output correctness, extraction scalability for large designs, and multi-result operation handling.

## Technical Changes

### 1. DAG-Aware S-Expression Output (Extraction-Gym)

**Problem:** For deeply shared e-graphs (e.g., `divider_32bit` with 472 nodes), naively inlining S-expressions for each reference caused exponential string blowup. A 472-node e-graph produced output estimated at exabytes, causing the process to hang or OOM.

**Solution:** Implemented a two-pass approach in `extract_result.rs`:

1. **Reference counting pass** (`count_refs`): walks the extracted DAG and counts how many times each e-class is referenced.
2. **DAG-aware builder** (`build_sexpr_dag`): for e-classes referenced more than once that represent actual MLIR operations (identified by `_` in the op name, matching `dialect_opname` convention), emits a `(let __sN <expr>)` definition and substitutes subsequent references with the alias `__sN`.

```rust
fn build_sexpr_dag(
    egraph: &EGraph,
    class_to_node: &HashMap<ClassId, NodeId>,
    class_id: &ClassId,
    shared: &HashSet<ClassId>,
    aliases: &mut HashMap<ClassId, String>,
    defs: &mut Vec<String>,
    alias_counter: &mut usize,
) -> String {
    if let Some(alias) = aliases.get(class_id) {
        return alias.clone();
    }
    // ... build child expressions recursively ...
    let full_expr = format!("({} {})", op_name, child_exprs.join(" "));
    if shared.contains(class_id) {
        let alias = format!("__s{}", *alias_counter);
        *alias_counter += 1;
        defs.push(format!("(let {} {})", alias, full_expr));
        aliases.insert(class_id.clone(), alias.clone());
        return alias;
    }
    full_expr
}
```

The `shared` set is filtered to only alias MLIR operations, not types, attributes, or vectors:

```rust
let shared: HashSet<ClassId> = ref_counts.iter()
    .filter(|(cid, &c)| {
        if c <= 1 { return false; }
        if let Some(nid) = class_to_node.get(*cid) {
            let node = &egraph[nid];
            !node.children.is_empty() && node.op.contains('_')
        } else { false }
    })
    .map(|(k, _)| k.clone())
    .collect();
```

This reduced `divider_32bit` output from hanging indefinitely to completing in under a second with kilobytes of output.

### 2. Extraction Timeout with Fallback (Extraction-Gym)

Added a `--timeout` CLI argument. When set, the primary extractor runs in a separate thread; if it exceeds the timeout, the process falls back to `faster-bottom-up`:

```rust
let result = if timeout_secs > 0 {
    let (tx, rx) = mpsc::channel();
    let eg = egraph.clone();
    let ext_name = extractor_name.clone();
    std::thread::spawn(move || {
        let r = run_extraction(&eg, &ext_name);
        let _ = tx.send(r);
    });
    match rx.recv_timeout(Duration::from_secs(timeout_secs)) {
        Ok(r) => r,
        Err(_) => {
            eprintln!("...timed out, falling back to faster-bottom-up");
            run_extraction(&egraph, "faster-bottom-up")
        }
    }
} else {
    run_extraction(&egraph, &extractor_name)
};
```

### 3. Nullary Function Wrapping (Extraction-Gym)

**Problem:** Leaf e-nodes representing 0-argument egglog functions (e.g., `UnitAttr`, `None`, type constructors like `I32`) were emitted as bare identifiers. DialEgg's `splitExpression` expected parenthesized S-expressions and failed with `Invalid expression: UnitAttr`.

**Solution:** Added `is_nullary_function` to detect leaf nodes that are egglog function calls (not numeric literals, not string literals) and wrap them in parentheses:

```rust
fn is_nullary_function(op: &str) -> bool {
    if op.is_empty() { return false; }
    let first = op.as_bytes()[0];
    if first.is_ascii_digit() { return false; }
    if first == b'-' && op.len() > 1 && op.as_bytes()[1].is_ascii_digit() { return false; }
    if first == b'"' { return false; }
    true
}
// In build_sexpr_dag, for leaf nodes:
} else if is_nullary_function(&node.op) {
    format!("({})", node.op)
} else {
    node.op.clone()
}
```

### 4. Shared Sub-Expression Parsing in Reconstruction (DialEgg)

**Problem:** The new `(let __sN ...)` output format needed to be consumed by `egg-opt --reconstruct-from-extraction`.

**Solution:** Added a pre-processing loop in `ReconstructFromExtractionPass::runOnBlock` that parses `(let ...)` lines and stores alias-to-operation mappings:

```cpp
while (extractedLines.back().rfind("(let __s", 0) == 0) {
    std::string_view letLine = extractedLines.back();
    auto split = Egglog::splitExpression(letLine);
    if (split.size() >= 3 && split[0] == "let") {
        std::string alias(split[1]);
        std::string_view exprStr = split[2];
        mlir::Operation* defOp = egglog.parseOperation(exprStr, builder);
        if (defOp != nullptr) {
            egglog.parsedOps[alias] = defOp;
        }
    }
    // ... read next line ...
}
```

`parseOperation` was also updated to resolve bare alias references:

```cpp
if (!newOpStr.empty() && newOpStr.front() != '(') {
    std::string key(newOpStr);
    auto it = parsedOps.find(key);
    if (it != parsedOps.end()) {
        return it->second;
    }
}
```

The `parsedOps` map key type was changed from `std::string_view` to `std::string` in `Egglog.h` to ensure proper ownership of alias strings:

```cpp
std::map<std::string, mlir::Operation*, std::less<>> parsedOps;
```

### 5. Multi-Result Operation Handling (DialEgg)

**Problem:** `hw.instance` produces multiple results, but `eggifyOpaqueOperation` created a single `(Value X (None))` entry, losing type information and result count.

**Solution:** For multi-result operations, eggify each result individually:

```cpp
EggifiedOp* Egglog::eggifyOpaqueOperation(mlir::Operation* op) {
    if (op->getNumResults() == 1) { return eggifyValue(op->getResult(0)); }
    if (op->getNumResults() == 0) { /* ... create (None) value ... */ }
    EggifiedOp* lastEggifiedOp = nullptr;
    for (unsigned i = 0; i < op->getNumResults(); i++) {
        lastEggifiedOp = eggifyValue(op->getResult(i));
    }
    return lastEggifiedOp;
}
```

### 6. Private HW Module Skipping (DialEgg)

**Problem:** When a design contains private `hw.module` submodules (e.g., for `hw.instance`), eggifying them overwrote the main module's e-graph JSON.

**Solution:** Both `EggifyOnlyPass` and `ReconstructFromExtractionPass` now skip private modules:

```cpp
if (auto hwModuleOp = llvm::dyn_cast<circt::hw::HWModuleOp>(&op)) {
    if (hwModuleOp.isPrivate()) { continue; }
    // ... process public module ...
}
```

### 7. Reconstruction Loop Fix & Result Count Validation (DialEgg)

**Problem:** A premature `break` in the reconstruction loop caused only the first extracted operation to be processed. Additionally, when equality saturation rules changed result counts, `replaceAllUsesWith` triggered an MLIR ValueRange assertion crash.

**Solution:** Removed the `break` and added explicit result count checking:

```cpp
if (newOp->getNumResults() == prevOp->getNumResults()) {
    prevOp->replaceAllUsesWith(newOp);
    prevOp->erase();
} else {
    llvm::errs() << "Warning: result count mismatch for " << eggOp->getPrintId()
                 << " (expected " << prevOp->getNumResults()
                 << ", got " << newOp->getNumResults() << "), skipping replacement\n";
    newOp->erase();
}
```

### 8. Extended Integer Type Declarations (DialEgg)

**Problem:** `sub_32bit` design uses non-standard integer widths (`Int19`, `Int21`, etc.) not declared in `base.egg`, causing `Unbound function Int19` during egglog execution.

**Solution:** Added comprehensive `IntN` type declarations to `src/base.egg` for widths 2-31 and selected larger widths up to 2048.

## Conclusion

This batch of fixes significantly improves the robustness of the Aspen extraction pipeline:

- **Scalability**: DAG-aware S-expression output eliminates exponential blowup for designs with shared sub-expressions, enabling extraction of 32-bit arithmetic circuits that previously timed out.
- **Correctness**: Nullary function wrapping, multi-result operation handling, and result count validation prevent crashes and silent miscompilation.
- **Resilience**: Extraction timeout with automatic fallback ensures the pipeline always produces a result, even for pathologically complex e-graphs.
- **Decoupled workflow**: The new `--eggify-only` and `--reconstruct-from-extraction` passes allow external extraction tools to be used between MLIR-to-egglog conversion and reconstruction.
